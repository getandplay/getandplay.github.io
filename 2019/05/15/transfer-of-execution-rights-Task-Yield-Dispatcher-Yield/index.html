Transfer of execution rights - Task.Yield, Dispatcher.Yield

Posted on 2019-05-15

Introduction

As the word Yield means, the Yield() method transfers excution right of current task, after that other tasks may get the right to excute. And the method of Yield() is existed in Task, Dispatcher, Thread class, and as the name shows, all of these methods is to transfer current task’s execution right.

Are there any differences ? How do they work?

In this article, we will see more details about Yield() methods.

Dispatcher.Yield

If we have to implement a time-consuming method, what methods or API will you choose to avoid affecting the UI thread? Maybe Invoke and InvokeAsync is a good choice, they split time-consuming task into small pieces that are executed at a lower priority than user input and rendering.

Dispatcher.Yield() does the same job, and its function is more familiar with Dispatcher.InvokeAsync than Dispatcher.Invoke.

It needs await before the method when we use it.

As the codes above, it splits the task to small pieces evetry time when excuting foreach method. Therefore, the UI thread can respond user interaction and rendering when running this time-consuming codes.

We can pass a parameter to Yield method when we use it, the value of this parameter is the priority of subsequent task. Its default value is DispatcherPriority.Background, and priority of Backgroud is lower than user’s input action’s priority (DispatcherPrority.Input) , UI’s priority (DispatcherPriority.Loaded) and rendering priority (DispatcherPriority.Render).

Task.Yield

We use Task.Yield method instead of Dispatcher.Yield in the example posted above.

This method has the same effect as Dispatcher.Yield(DispatcherPriority.Normal). Task depends on SynchronizationContext to schedule threads, and the WPF UI thread’s SynchronizationContext is set to DispatcherSynchronizationContext , it relies Dispatcher to shedule; When creating an instance of DispatcherSynchronizationContext its default priority is Normal, and WPF dosen’t pass a special value. So, after calling Task.Yield on WPF UI thread to transfer excution right, it uses Normal priority to recover. Therefore, its effect is the same as Dispatcher.Yield(DispatcherPriority.Normal).

Due to Task.Yield‘s Normal priority, its effect on the UI thread is not as good as Dispatcher.Yield. But, it is important to note that SynchronizationContext is not related to Dispatcher , so Task.Yield can be called on everty thread. And it will be a good choice to use Task.Yield to split time-consuming task into small task in Task method.
